直接量
  • 缺点：需要创建多个 Car 的实例
new Object
  • 缺点：需要创建多个 Car 的实例
工厂方式 + 工厂函数外定义对象的方法
  • 优点：每个对象的同一个函数，只会创建一次
  • 缺点：看起来不像对象的创建方式
自定义构造函数方式
  • 优点：看起来像对象的创建方式
  • 缺点：每个对象的同一个函数，会各创建一次
原型方式
  • 优点：每个对象的同一个函数，只会创建一次
  • 缺点：
    1. 不能通过传参来初始化属性值
    2. 当属性值是引用会产生问题（如右注释）
混合的构造函数/原型方式 （推荐）
  • 好处：
  1. 用构造函数定义对象的所有非函数属性，
    每个对象都具有自己的对象属性实例
  2. 用原型方式定义对象的函数属性，
    每个对象的同一个函数，只会创建一次
动态原型方法 （推荐）
  • 好处：看起来更像其他语言中的类定义
